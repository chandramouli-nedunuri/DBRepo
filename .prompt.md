# GitHub Copilot Agent Rules

## Mandatory Database Operation Rules

### Rule #1: NO DELETE Operations
**DO NOT delete or run delete command on any content from any Database.**
- Never execute DELETE statements on any database
- Never execute DROP TABLE, DROP CONSTRAINT, or similar destructive commands
- Instead: Provide DELETE queries to the user with clear documentation
- User must execute these queries manually on the database

### Rule #2: NO MODIFY/UPDATE/ALTER Operations  
**DO NOT modify, update, or change any existing data in the database.**
- Never execute UPDATE statements that modify existing records
- Never execute ALTER statements that change existing data
- Never execute INSERT statements that add new data
- Instead: Provide UPDATE, INSERT, or ALTER commands to the user
- User must execute these commands manually on the database

### Rule #3: Authorization Scope
**You are authorized to execute ONLY CREATE and SELECT statements on any database.**
- ✓ CREATE statements (CREATE TABLE, CREATE SCHEMA, CREATE INDEX, CREATE PROCEDURE, etc.)
- ✓ SELECT statements (queries, diagnostics, data retrieval)
- ✗ DELETE statements (forbidden)
- ✗ UPDATE statements (forbidden)
- ✗ DROP statements (forbidden)
- ✗ ALTER statements that modify existing data (forbidden)

## Implementation Guidelines

1. **For Destructive Operations**: Always ask the user first and provide the query text for manual execution
2. **For Data Modifications**: Provide the command/query and ask the user to execute it manually
3. **For Safe Operations**: CREATE and SELECT statements may be executed directly
4. **For Connectivity Testing**: SELECT statements for diagnostics are allowed
5. **For FK Constraint Drops**: Provide DROP CONSTRAINT queries for user manual execution; do NOT auto-execute

## Exception Cases
- Migration setup scripts containing CREATE statements may be executed
- System diagnostics via SELECT are permitted
- Connection testing via SELECT @@VERSION is permitted
